# Med-Recallix 需求文档

> Recallix Med — 医考记忆强化个人助手

## 1. 背景与目标

### 1.1 业务背景

执业医师 / 助理医师资格考试涵盖内科、外科、妇产、儿科、基础医学等众多学科，知识点体量庞大。备考者普遍面临"学了就忘"的问题，缺乏系统性的记忆管理工具。传统刷题 App 侧重题库，而非基于记忆科学的个性化复习规划。

### 1.2 目标

- **业务目标**：为备考者提供基于遗忘曲线（Ebbinghaus）的智能复习调度，提升长期记忆留存率
- **用户目标**：解决"学了忘、忘了学"的恶性循环，让复习有节奏、有反馈、有陪伴感
- **产品定位**：AI 学习伙伴 — 不只是工具，更是懂你的备考搭子

## 2. 用户与角色

| 角色 | 描述 | 主要诉求 |
|------|------|----------|
| 备考者（主人） | 执业医师/助理医师资格考试备考者 | 高效记忆、科学复习、有人督促 |
| Med-Recallix（AI 伙伴） | 具有人格设定的 AI 学习伙伴 | 陪伴学习、主动提醒、出题检测 |

### 2.1 AI 伙伴人格设定

- **性格**：温暖但严格、鼓励式沟通、偶尔幽默
- **称呼**：称用户为"主人"
- **行为**：主动提醒复习、根据表现调整难度、给予正向反馈

## 3. 功能需求

### 3.1 User Stories

#### US-001: 备考者可以录入知识点，以便建立个人知识库
**优先级**：P0
**验收标准**：
1. Given 用户在知识录入页, When 输入知识点标题+内容+所属科目, Then 知识点保存成功并可检索
2. Given 已有知识点, When 用户编辑内容, Then 更新成功且不丢失学习记录

#### US-002: 备考者可以让 AI 根据知识点生成选择题，以便自我检测
**优先级**：P0
**验收标准**：
1. Given 用户选择一个知识点, When 请求 AI 出题, Then 返回 1-5 道 A1/A2 型选择题
2. Given AI 生成的题目, When 题目内容不准确, Then 用户可标记/反馈并重新生成

#### US-003: 备考者可以根据记忆曲线安排复习，以便科学抗遗忘
**优先级**：P0（MVP 核心）
**验收标准**：
1. Given 用户完成一个知识点学习, When 首次学习完成, Then 系统按 SM-2 算法计算下次复习时间
2. Given 到达复习时间, When 用户打开应用, Then 待复习知识点按紧急度排列展示
3. Given 用户完成复习并自评, When 评分为 0-2 (困难), Then 缩短下次复习间隔
4. Given 用户完成复习并自评, When 评分为 3-5 (容易), Then 延长下次复习间隔

#### US-004: 备考者可以在应用内收到复习提醒，以便不错过最佳复习时机
**优先级**：P0（MVP 核心）
**验收标准**：
1. Given 用户登录应用, When 有待复习知识点, Then 首页醒目展示待复习数量和紧急程度
2. Given 有逾期未复习的知识点, When 用户打开应用, Then AI 伙伴以对话方式提醒并鼓励

#### US-005: 备考者可以与 AI 伙伴对话，以便获得学习陪伴和答疑
**优先级**：P1
**验收标准**：
1. Given 用户在聊天界面, When 发送医学问题, Then AI 伙伴以学习伙伴语气回答
2. Given 对话进行中, When AI 伙伴回复, Then 支持流式输出，实时展示打字效果
3. Given 用户关闭应用再打开, When 进入聊天, Then 可查看历史对话记录

#### US-006: 备考者可以查看学习统计，以便了解自己的学习进度
**优先级**：P1
**验收标准**：
1. Given 用户在统计页, When 查看概览, Then 展示已学/待学/已掌握知识点数量
2. Given 用户在统计页, When 查看记忆曲线, Then 展示各知识点的记忆保持率趋势

#### US-007: 备考者可以通过简单认证登录，以便保护个人数据
**优先级**：P0
**验收标准**：
1. Given 新用户, When 输入用户名+密码注册, Then 创建账户成功
2. Given 已注册用户, When 输入正确凭据, Then 登录成功并恢复学习数据
3. Given 登录状态, When Token 过期, Then 自动提示重新登录

## 4. 非功能需求

### 4.1 性能
- 页面首屏加载 < 2s（EdgeOne CDN 边缘加速）
- AI 出题响应首 token < 1s（流式输出）
- KV 读取延迟 < 100ms

### 4.2 安全
- 密码 bcrypt 哈希存储（不可明文）
- AI API Key 存储在 KV 命名空间，不暴露给前端
- JWT Token 认证，HttpOnly Cookie

### 4.3 可用性
- 移动端优先，支持响应式布局
- 离线时显示缓存的知识点和复习计划（IndexedDB）
- 弱网环境下优雅降级

### 4.4 存储约束
- KV 免费版 1GB 总容量
- 关键数据存 KV：用户信息、知识点元数据、SM-2 调度参数、AI Key 配置
- 大体量数据存浏览器 IndexedDB：对话历史、题目缓存、离线知识点内容
- 数据结构精简设计，单用户预估 < 50MB KV 用量

## 5. Scope 定义

### In Scope（本次做）
- SM-2 间隔复习引擎 + 复习仪表盘
- 知识点 CRUD（树状分类 + 标签）
- AI 选择题生成（A1/A2 型）
- 做题 + 自评 + 记忆调度更新
- 用户名密码认证
- 应用内复习提醒
- 移动端优先 UI
- PWA 支持（可安装到桌面、离线缓存、standalone 模式）

### Out of Scope（后续迭代）
- AI 对话伙伴（P1，第二期）
- 学习统计面板（P1，第二期）
- 知识点回溯时间线（P1，第二期）
- 微信推送/邮件提醒
- 多端数据同步（PWA）
- 题库导入（Excel/PDF）
- 多人协作/分享
- A3/A4/B 型题等复杂题型

### 假设与依赖
- **假设**：用户具备基础医学知识，能理解医学术语
- **假设**：单实例部署，用户量 < 100（KV 1GB 免费额度内）
- **依赖**：腾讯云 EdgeOne Pages（部署 + KV 存储 + Edge Functions）
- **依赖**：Kimi-for-coding API（OpenAI 兼容，免费额度）
- **依赖**：SM-2 算法库（`supermemo` 或 `@open-spaced-repetition/sm-2`）

## 6. Edge Cases 与异常处理

| 场景 | 输入/条件 | 期望行为 |
|------|-----------|----------|
| AI API 不可用 | 网络异常或 API Key 失效 | 显示友好错误提示，允许手动出题 |
| KV 存储满 | 接近 1GB 上限 | 提示存储空间不足，建议清理旧数据 |
| 长时间未复习 | 超过 30 天未登录 | 重新调整所有知识点到"重新学习"状态 |
| 空知识库 | 新用户首次使用 | 引导式 onboarding，提供示例知识点 |
| AI 生成低质量题目 | 医学内容不准确 | 用户可标记"有误"，触发重新生成 |
| 密码遗忘 | 无邮箱/手机绑定 | MVP 暂不支持找回，提示联系管理员 |
| 并发复习 | 多端同时操作 | KV 最终一致性，以最后写入为准 |
| 超长知识点内容 | 内容 > 10KB | 前端截断展示，KV 按完整内容存储 |

## 7. 验收标准（Definition of Done）

### MVP 验收清单
- [ ] 用户可注册/登录，数据隔离
- [ ] 可创建/编辑/删除知识点，支持分类和标签
- [ ] AI 可根据知识点生成 A1/A2 选择题
- [ ] 答题后可自评难度，SM-2 算法自动计算下次复习时间
- [ ] 首页仪表盘展示今日待复习知识点
- [ ] 逾期知识点有醒目提醒
- [ ] 移动端 UI 可用且流畅
- [ ] 成功部署到 EdgeOne Pages
- [ ] AI API Key 安全存储在 KV，不泄露
