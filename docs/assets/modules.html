Med-Recallix 功能模块详解

6 大功能模块 × 14 个 API 端点 × 9 个页面

说明：下表将「同一路径的多种 HTTP 方法」合并为一条端点行（如 GET, POST /api/knowledge），共 14 条；按单次 HTTP 调用计为 16 次。配置、档案、统计等辅助接口见折叠附表。

1. 模块全景

六大模块的数据流与协作关系如下：auth 为全局提供 userId；knowledge 驱动 review 与 quiz；agent 为 chat 构建上下文；review 与 agent 通过 episode 等机制联动。

2. 认证模块（Auth）

文件与类型

  • 文件：auth.service.ts、auth.schema.ts、use-auth.ts、components/
  • API：POST /api/auth/register、POST /api/auth/login、GET /api/auth/me
  • 关键类型：AuthUser、StoredUser、JWTPayload
流程概要：注册 → PBKDF2 哈希 → 写入 KV → JWT 签发 → HttpOnly Cookie。
  • 安全：PBKDF2，迭代 100,000；盐 16 字节；摘要算法 SHA-256。
示例：认证数据流（伪代码） 
// 注册：密码不落盘明文
hash = PBKDF2(password, salt, 100000, SHA-256)
kv.put(`user:${username}`, { hash, salt, ... })
// 登录成功：Set-Cookie: token=jwt.sign(JWTPayload)

3. 知识点模块（Knowledge）

CRUD · 索引 · 分类 · 标签

  • 文件：knowledge.service.ts、knowledge.schema.ts、components/
  • API：GET/POST /api/knowledge；GET/PUT/DELETE /api/knowledge/:id；GET /api/knowledge/:id/recall
  • 能力：按用户隔离的 CRUD、索引、分类树、标签体系。
  • KV 键示例：kp_{userId}_{id}、kp_index_{userId}、cat_{userId}
  • 联动：创建知识点时自动创建对应复习卡片。
KV 键与索引（示意） 
// 单条 KP 文档
kp_{userId}_{kpId} → KnowledgePoint JSON
// 用户侧 ID 列表 / 分类
kp_index_{userId} → string[]
cat_{userId}      → CategoryTree JSON

4. 复习模块（Review）— SM-2 核心

  • 文件：review.service.ts、sm2.ts、components/review-timeline.tsx
  • API：GET /api/cards、PUT /api/cards/:id
  • SM-2：间隔（interval）、重复次数（repetition）、难度因子（efactor）、评分档 0–5。
  • 卡片生命周期：新卡片 → 学习中 → 复习中 → 已掌握。
  • 牌组存储：用户全部卡片聚合在同一 KV 文档中管理。
  • 连续打卡：统计连续学习日（streak）。
  • 复习历史：每张卡片保留最近约 50 条回溯记录。
SM-2 评分与调度（概念） 
// grade 0–5 驱动 interval / repetition / efactor 更新
nextReviewAt = now + schedule(interval, efactor, grade)
// 时间线组件读取最近 history[] 展示

5. AI Agent 模块

  • 文件：soul.ts、context-builder.ts、profile.service.ts、memory.service.ts、episode.service.ts
  • 人格（AGENT_SOUL）：称呼用户为「主人」，医学助理角色定位。
  • 上下文构建：学习档案 + 记忆检索 + 当日 episode → 拼入 system prompt。
  • 记忆：上限约 100 条；关键词召回；重要性打分。
  • 档案：考试目标、科目强弱、学习偏好等。
  • Episode：按日汇总学习行为（复习次数、时长、测验分数等）。
上下文拼装顺序（示意） 
system =
  AGENT_SOUL
  + formatProfile(profile)
  + recallMemories(query, max: 100)
  + todayEpisodeSummary(episode)

6. 对话模块（Chat）

  • 文件：chat.service.ts、use-chat.ts、components/
  • API：GET/POST /api/chat；GET/DELETE /api/chat/:sessionId
  • 流式输出：Vercel AI SDK streamText，注入 Agent 上下文。
  • 会话：创建、列表、删除、消息历史加载。
  • 回复后任务：记忆抽取、会话压缩（compaction）等。
  • 引导：新用户 onboarding 流程。
流式对话与后置任务 
streamText({ system: builtContext, messages })
// 结束后：extractMemoryChunks(), maybeCompactSession()

7. 出题模块（Quiz）

  • 文件：quiz.service.ts、quiz.prompts.ts
  • API：POST /api/quiz/generate
  • 流程：选择知识点 → 构造 prompt → 模型生成五选一 MCQ（A–E）→ 解析 JSON → QuizQuestion[]。
  • 特性：无持久化题库，请求级无状态生成。

8. API 端点总览

六大模块 · REST

方法 路径 需登录 说明
POST/api/auth/register否用户注册
POST/api/auth/login否登录并下发 JWT Cookie
GET/api/auth/me是当前用户信息
GET, POST/api/knowledge是列表 / 新建（新建联动复习卡片）
GET/api/knowledge/:id是知识点详情
PUT/api/knowledge/:id是更新知识点
DELETE/api/knowledge/:id是删除知识点
GET/api/knowledge/:id/recall是复习回溯 / 召回数据
GET/api/cards是获取牌组与待复习卡片
PUT/api/cards/:id是评分并更新 SM-2
GET, POST/api/chat是会话列表 / 创建或流式发消息
GET/api/chat/:sessionId是加载会话历史
DELETE/api/chat/:sessionId是删除会话
POST/api/quiz/generate是基于所选 KP 生成 MCQ（A–E）
附：辅助接口（非六大模块表内统计）
方法路径需登录说明
GET/api/profile是学习档案
PUT/api/profile是更新档案
GET/api/config视实现读取配置（如 AI）
PUT/api/config视实现写入配置
GET/api/stats是学习统计
