Med-Recallix 系统技术架构

医学考试记忆辅助学习系统 — 全栈技术架构解析

一句话：基于 SM-2 间隔重复算法与 AI Agent 的医学考试记忆辅助学习 PWA，前后端一体部署于 EdgeOne Pages，数据经 Edge Function 代理访问分布式 KV。

1. 项目概览

面向医学考试场景，将间隔复习、知识点管理与对话式 AI 助手整合为可安装的 Web 应用，降低运维成本并贴近全球边缘访问。

层级技术栈
前端Next.js 16 + React 19 + TypeScript + Tailwind CSS v4
后端Next.js API Routes（Node.js Runtime）
AIVercel AI SDK + OpenAI 兼容 API（Moonshot / DeepSeek 等）
存储EdgeOne Pages KV（分布式键值存储）
部署EdgeOne Pages（Edge CDN + Functions）
PWAService Worker + Web App Manifest

  • 展现层：App Router 页面与可复用组件，负责交互与展示，尽量保持轻逻辑。
  • API 层：REST 风格路由处理器，解析请求、校验会话、编排服务调用。
  • 服务层：按领域划分的业务模块，封装复习算法、对话、出题与 Agent 行为。
  • 基础设施层：KV 持久化、AI 流式调用与运行时配置读取，对上提供稳定接口。

6. 核心技术选型

为何采用 Next.js 16（App Router、RSC、Edge）

统一技术栈减少上下文切换；App Router 与 RSC 利于路由与数据边界清晰；可按需在 Edge 与 Node 运行时之间取舍，适配 Pages 部署模型。

为何采用 EdgeOne Pages KV（Serverless、免运维、全球分布）

无需自建数据库集群，按量扩展；键值模型契合用户配置与业务 JSON 文档；与 Edge Function 同域集成，通过代理安全访问。

为何 SM-2 + SuperMemo 思路（经典算法、轻量）

医学记忆场景验证充分；实现体量小、易测试；与产品「复习卡片」模型天然契合。

为何 Vercel AI SDK（流式、多厂商）

统一流式响应与工具调用抽象；可切换 Moonshot、DeepSeek 等 OpenAI 兼容端点，降低供应商锁定感知。

为何 PWA（跨端、可安装、可离线能力）

医学生多用手机碎片学习；安装到主屏幕接近原生体验；Service Worker 为静态与缓存策略提供基础（具体离线范围以实现为准）。

7. 安全架构

  • JWT + HttpOnly Cookie：令牌不暴露给页面脚本，降低 XSS 窃取风险。
  • PBKDF2 密码哈希：约 10 万次迭代、SHA-256，拖慢离线暴力破解。
  • Middleware 鉴权闸门：在路由层统一拦截未登录访问受保护资源。
  • KV 代理与密钥保护：敏感配置经 Edge Function / 服务端访问 KV，避免前端直连与密钥泄露。
