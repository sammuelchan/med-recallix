# Med-Recallix 测试用例

## 约定

### 文件组织

按功能模块拆分，每个 `.md` 文件对应一个独立模块：

| 文件 | 模块 | 阶段 |
|------|------|------|
| [auth.md](./auth.md) | 认证（注册/登录/JWT） | MVP |
| [knowledge.md](./knowledge.md) | 知识点管理 | MVP |
| [review.md](./review.md) | SM-2 间隔重复复习 | MVP |
| [quiz.md](./quiz.md) | AI 出题 | MVP |
| [settings.md](./settings.md) | 设置与 KV 配置 | MVP |
| [pwa.md](./pwa.md) | PWA / Service Worker / 部署 | MVP |
| [chat.md](./chat.md) | AI 对话 + Agent 记忆 | P1 |
| [stats.md](./stats.md) | 学习统计 + 知识回溯 | P1 |

### 测试用例格式

每条用例包含：

```markdown
### TC-{MODULE}-{NNN}: {标题}

**优先级**：P0 / P1 / P2
**关联**：US-{NNN} / T{NNN}
**前置条件**：
- ...

**步骤**：
1. ...
2. ...

**预期结果**：
- ...

**状态**：⏳ 未实现 | 🔲 待测试 | ✅ 通过 | ❌ 失败 | 🚫 跳过
```

### 状态说明

| 状态 | 含义 | 适用场景 |
|------|------|---------|
| ⏳ 未实现 | 功能尚未开发，用例为规格定义 | P1 功能、待开发的 gap |
| 🔲 待测试 | 功能已实现，用例待人工验证 | MVP 功能回归 |
| ✅ 通过 | 已验证通过 | 回归通过 |
| ❌ 失败 | 验证未通过，需修复 | Bug |
| 🚫 跳过 | 暂不测试（环境限制等） | 需线上环境 |

### 工作流

```
发现功能缺漏 → 新增 ⏳ 测试用例 → 开发功能 → 标记 🔲 → 手动验证 → 标记 ✅ / ❌
                ↑                                                    |
                └────────────────── 持续演进 ←───────────────────────┘
```

### 给 AI 的指引

AI agent 扫描本目录时，应：
1. 读取所有 `.md` 文件
2. 识别状态为 `⏳ 未实现` 的用例 → 这些是待开发的功能规格
3. 识别状态为 `❌ 失败` 的用例 → 这些是需要修复的 bug
4. 开发完成后将状态更新为 `🔲 待测试`
5. 不要修改 `✅ 通过` 的用例状态（除非代码变更影响了该功能）
