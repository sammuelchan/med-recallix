# 知识点管理模块测试用例

> 模块路径：`src/modules/knowledge/`
> API：`/api/knowledge` (GET/POST), `/api/knowledge/[id]` (GET/PUT/DELETE)
> 关联需求：US-001

---

## 创建知识点

### TC-KP-001: 正常创建知识点

**优先级**：P0
**关联**：US-001, T501
**前置条件**：
- 用户已登录

**步骤**：
1. 进入 `/knowledge/new` 页面
2. 输入标题："心力衰竭的分级"
3. 输入内容："NYHA 分级：I级 无症状...（详细内容）"
4. 选择学科："内科"
5. 选择系统："循环系统"
6. 点击"保存"按钮

**预期结果**：
- 知识点创建成功
- 跳转到知识点列表页
- 列表中出现新创建的知识点
- KV 中存在 `kp_{userId}_{kpId}` 记录
- KV 中 `kp_index_{userId}` 包含新条目
- 自动创建一张 SM-2 复习卡片（`deck_{userId}` 中新增 card）

**状态**：🔲 待测试

## API 层

### TC-KP-040: POST /api/knowledge 创建成功

**优先级**：P0
**关联**：T502
**前置条件**：
- 携带有效 Cookie

**步骤**：
1. 发送 `POST /api/knowledge`，body:
   ```json
   {
     "title": "API创建测试",
     "content": "知识点内容",
     "subject": "内科",
     "system": "呼吸系统"
   }
   ```

**预期结果**：
- 返回 201
- body 包含完整的 `KnowledgePoint` 对象（含 id, createdAt）

**状态**：✅ 通过
