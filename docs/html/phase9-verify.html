Step 0: 配置 API Key

Phase 9 的 AI 对话功能需要有效的 API Key。以下 3 种方式任选其一：

方式 A: 设置页面（推荐）

  1. 打开 http://localhost:3000
  2. 注册/登录后进入 设置 页
  3. 填入 API Key、Base URL、Model
  4. 点击保存

数据自动写入 ~/.med-recallix/kv/med_config/ai_config.json

方式 B: 手动写文件

# Windows PowerShell
$dir = "$env:USERPROFILE\.med-recallix\kv\med_config"
New-Item -ItemType Directory -Force -Path $dir

# 写入 AI 配置
@'
{
  "apiKey": "sk-你的真实Key",
  "baseURL": "https://api.kimi.com/coding/v1",
  "model": "kimi-for-coding"
}
'@ | Out-File -Encoding utf8 "$dir\ai_config.json"
方式 C: 通过 .env.local 环境变量 
# .env.local
KIMI_API_KEY=sk-你的真实Key
AI_BASE_URL=https://api.kimi.com/coding/v1
AI_MODEL=kimi-for-coding
JWT_SECRET=local-dev-secret-change-in-production

修改后需重启 pnpm dev。

Kimi API Key 获取

前往 Moonshot AI 控制台 创建 API Key。kimi-for-coding 模型面向开发者免费。

如果使用其他 OpenAI 兼容模型，只需修改 baseURL 和 model。

EdgeOne 部署指引

本地验证通过后，按以下步骤推送到 EdgeOne 生产环境。

1推送代码到 Git 仓库
cd c:\workspace\my-workspace\med-recallix
git add -A
git commit -m "feat: Phase 9 - AI 对话 + Agent 记忆系统"
git remote add origin https://github.com/你的用户名/med-recallix.git
git push -u origin main
2在 EdgeOne Pages 创建项目
  1. 登录 EdgeOne Pages 控制台
  2. 点击「创建项目」→「导入 Git 仓库」
  3. 授权 GitHub，选择 med-recallix
  4. 框架选 Next.js，构建命令 pnpm build，输出 .next
  5. 点击部署
3配置 KV 存储
  1. 顶部导航 → KV 存储 → 创建两个命名空间
  2. 项目详情 → KV 存储 → 绑定：
    • 第一个命名空间 → 变量名 MED_CONFIG
    • 第二个命名空间 → 变量名 MED_DATA
  3. 进入 MED_CONFIG 命名空间 → 新建记录：
KeyValue
ai_config {"apiKey":"sk-你的Key","baseURL":"https://api.kimi.com/coding/v1","model":"kimi-for-coding"}

JWT_SECRET 已内置在 .env.example 中，构建时自动 copy 到 .env，无需额外配置。

4重新部署并验证

绑定 KV 后点击「重新部署」。构建时会自动执行 cp .env.example .env，JWT_SECRET 从 .env.example 注入。部署完成后用分配的域名重复上述 4 条 Journey。
