Med-Recallix 部署验收指南

从 Git Push 到生产验收的完整流程 — 预计耗时 30 分钟

核心要点

  • 部署方式：Git Push → EdgeOne 自动构建部署（无需手动上传）
  • 必须配置：KV 存储绑定 + AI API Key 写入 KV
  • 验收标准：7 条 User Journey 全部通过即为验收合格

第一部分：部署到 EdgeOne

flowchart LR A[Git Push] --> B[EdgeOne 构建] B --> C[静态资源 CDN] B --> D[Edge Functions] D --> E[KV 存储] D --> F[AI API] C --> G[用户访问] D --> G

Step 1：推送代码到 GitHub

git add -A
git commit -m "feat: complete MVP + P1 features"
git push origin main
⚠️ 确认清单
  • .env.example 中有 JWT_SECRET 值
  • edgeone.json 存在且安装命令为 pnpm install、构建命令为 cp .env.example .env && pnpm run build
  • .gitignore 包含 .env.local 和 .next/

Step 2：创建 EdgeOne Pages 项目

1
登录 EdgeOne Pages 控制台 → 点击 创建项目
2
选择 导入 Git 仓库 → 授权 GitHub → 选择 med-recallix
3
构建设置自动读取 edgeone.json，确认各项值正确后点击 部署
设置项值
框架预设Next
根目录./
输出目录.next
构建命令cp .env.example .env && pnpm run build
安装命令pnpm install

⚠️ 安装命令和构建命令是分开填写的。cp .env.example .env 放在构建命令中， 确保构建时 JWT_SECRET 等环境变量从 .env.example 注入。

4
等待构建完成（约 2-5 分钟），获取分配域名如 med-recallix-xxx.edgeone.app

Step 3：配置 KV 存储

关键步骤

没有 KV 配置，所有 API 将返回 500 错误。

1
开通 KV 服务：控制台顶部 → KV 存储 → 立即申请
2
创建 2 个命名空间：
命名空间名称用途
med_config系统配置
med_data业务数据
3
绑定到项目：项目详情 → KV 存储 → 绑定命名空间
命名空间变量名（必须一致）
med_configMED_CONFIG
med_dataMED_DATA
4
写入 AI 配置：进入 med_config 命名空间 → 记录管理 → 新建记录 
Key:   ai_config
Value: {"apiKey":"sk-你的API密钥","baseURL":"https://api.moonshot.cn/v1","model":"moonshot-v1-auto"}

支持 Moonshot / DeepSeek / OpenAI 等任何 OpenAI 兼容 API

5
重新部署：KV 绑定后必须重新部署才能生效。进入项目 → 部署 → 重新部署
⚠️ 常见陷阱
  • 变量名 MED_CONFIG 和 MED_DATA 大小写敏感，写错会导致 500 错误
  • KV 绑定后不重新部署 = 绑定不生效
  • AI API Key 不要放在 .env 文件中，必须通过 KV 或设置页面配置

第二部分：验收测试

以下 7 条 User Journey 覆盖全部 MVP + P1 功能。逐条执行，全部通过即验收合格。

Journey 1：注册与登录

认证系统

  • 访问部署域名，看到登录页面
  • 点击注册，输入用户名 test01、密码 Test1234，注册成功
  • 自动跳转到首页（Dashboard）
  • 刷新页面，仍保持登录状态（JWT 有效）
  • 点击设置 → 退出登录 → 回到登录页
  • 用刚注册的账号重新登录成功
失败排查
现象原因修复
注册 500 错误KV MED_DATA 未绑定检查 KV 绑定变量名
登录后刷新变登录页JWT_SECRET 未注入确认 .env.example 有值且构建命令含 cp

Journey 2：知识点管理

知识点 CRUD

  • 底部导航 → 知识点 → 右上角 + 新建
  • 填写标题「心力衰竭分类」，分类「内科/心血管」，内容随意，保存
  • 列表中出现该知识点
  • 点击进入详情，内容正确显示
  • 点击编辑图标，修改标题 → 保存 → 标题更新
  • 再新建 2 个知识点，列表显示 3 个

Journey 3：SM-2 复习

间隔重复引擎

  • 知识点详情中看到「复习回溯」区域（应显示「尚未加入复习卡片」）
  • 底部导航 → 复习 → 看到待复习卡片（新建的知识点自动生成卡片）
  • 开始复习，对卡片评分（0-5），提交后卡片更新
  • 返回首页，看到复习统计更新（待复习数/已完成数）
  • 连续天数横幅出现
  • 回到知识点详情 → 复习回溯时间线出现刚才的评分记录

Journey 4：AI 出题

AI 集成

  • 确认设置页面中 API Key 已配置（或 KV 中已写入）
  • 底部导航 → 复习页面 → 找到 AI 出题入口
  • 选择知识点 → 生成题目 → AI 返回选择题
  • 答题提交 → 显示正确答案和解析
AI 错误排查
提示原因修复
尚未配置 AI API KeyKV 没有 ai_config 或 apiKey 为空在 KV med_config 或设置页面配置
AI 账户余额不足Moonshot 账户欠费充值或更换 API
模型不支持 Web 调用误用了 kimi-for-coding改为 moonshot-v1-auto

Journey 5：AI 对话 + Agent 记忆

P1 核心功能

  • 底部导航 → 对话
  • 首次引导：如果是新用户，看到欢迎界面，发送「你好」
  • AI 以引导模式回复，收集学习档案（考试目标、强弱科目等）
  • 配合 AI 完成档案收集，AI 输出 JSON 后自动保存
  • 开始正常对话，问一个医学问题如「糖尿病分型的鉴别要点」
  • AI 流式回复，打字效果正常，称呼「主人」
  • 左上角打开会话列表 → 看到对话记录
  • 新建对话 → 新会话正常工作
  • 切换回旧会话 → 历史消息正确加载

Journey 6：学习统计

P1 数据可视化

  • 首页右上角点击柱状图图标 → 进入统计页面
  • 看到连续天数横幅（火焰图标）
  • 看到核心数据（知识点数/已掌握/今日待复习）
  • 掌握进度条显示正确百分比
  • 7 天复习量柱状图正常渲染
  • 今日学习区域显示学习时长和复习卡片数

Journey 7：PWA 安装

渐进式 Web 应用

  • 手机浏览器访问部署域名
  • 界面移动端适配良好（底部导航、卡片布局）
  • 浏览器菜单 → 添加到主屏幕
  • 主屏幕出现 App 图标
  • 点击图标启动 → 全屏显示，无浏览器地址栏
